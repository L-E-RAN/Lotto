# 🎰 LottoStat AI — ניתוח סטטיסטי של הגרלות הלוטו בישראל

מערכת ווב מלאה בעברית (RTL) למשיכה, שמירה וניתוח סטטיסטי של היסטוריית הלוטו הישראלי
(6 מתוך 37 + מספר חזק 1–7), כולל מנוע תחזיות מבוסס מודלים סטטיסטיים ומנגנון **Backtesting**.

> ⚠️ **הבהרה חשובה:** התחזית מבוססת על ניתוח סטטיסטי של נתוני עבר בלבד.
> אין אפשרות להבטיח או לנבא תוצאות לוטו. השימוש במערכת הוא לצורכי ניתוח ובידור בלבד.
> נתוני עבר אינם מנבאים תוצאות עתידיות.

---

## ✨ יכולות

- **דשבורד** — הגרלה אחרונה, תחזית הבאה, מספרים חמים/קרים/מאחרים, ביצועי מודל, גרף השוואה מול אקראי.
- **הגרלות** — טבלה מלאה, חיפוש לפי תאריך/מספר, סינון לפי שנה, עדכון ידני, ייצוא CSV.
- **סטטיסטיקת מספרים** — הופעות, אחוז, איחור, חלונות 10/25/50/100, מגמה וציון, מיון בלחיצה.
- **מספר חזק** — תדירות, חם/קר, תחזית, גרף.
- **זוגות ושלשות** — נפוצים, מאחרים, Heatmap.
- **תבניות** — זוגי/אי-זוגי, נמוך/גבוה, סכום, רצפים, פיזור, עשורים, זיהוי "תאריכי לידה", וכלי לניתוח צירוף עצמאי.
- **תחזית הבאה** — 3 תחזיות (ראשית / מאוזנת / אגרסיבית), נימוק לכל מספר, אחוזי פגיעה היסטוריים והשוואה מול אקראי.
- **Backtesting** — הרצת בדיקת עבר walk-forward לכל מודל מול בחירה אקראית.

## 🧠 מנוע התחזית — מודלים

| מודל | תיאור |
|------|-------|
| `hot_numbers_model` | ניקוד למספרים שהופיעו הרבה לאחרונה |
| `cold_numbers_model` | ניקוד למספרים שלא הופיעו זמן רב (ללא הבטחה שייצאו) |
| `weighted_model` | משוקלל: תדירות 25% · עדכניות 25% · איחור 20% · זוגות 15% · תבנית 15% |
| `pair_model` | התאמה לפי זוגות/שלשות היסטוריים |
| `anti_popularity_model` | מעדיף צירופים לא-אנושיים (לא 1–31 בלבד, לא רצפים, לא סימטריה) |
| `random_filtered_model` | אלפי צירופים אקראיים, מסוננים לפי תבניות הגיוניות ומדורגים |

**כללי סינון לתחזית:** 2–4 זוגיים, 2–4 נמוכים, ≤2 מאותו עשור, ≤1 רצף, לפחות מספר אחד מעל 31,
סכום בטווח הנפוץ היסטורית, ולא צירוף שכבר הופיע בעבר.

## 🛠️ טכנולוגיות

Frontend: **React + Vite + Recharts** · Backend: **Node.js + Express** · DB: **SQLite (better-sqlite3)** ·
תזמון: **node-cron** · משיכת נתונים: **Axios + Cheerio**.

## 🚀 התקנה והרצה (פקודה אחת)

דרישה מקדימה: **Node.js 18+**.

```bash
# מתיקיית C:\Lotto
npm install      # מתקין שורש + server + client אוטומטית
npm run dev      # מריץ שרת (3001) ולקוח (5173) במקביל
```

פתחו דפדפן בכתובת: **http://localhost:5173**

> בהרצה הראשונה, אם אין נתונים, המערכת מבצעת **seed** אוטומטי של היסטוריית הגרלות לפיתוח,
> יוצרת תחזית ראשונית, ומחשבת ביצועי מודלים ברקע.

### פקודות נוספות

```bash
npm run seed     # יצירת נתוני seed ידנית (אם המאגר ריק)
npm run build    # build לפרודקשן של הלקוח (השרת יגיש אותו אוטומטית)
```

לפרודקשן: `npm run build` ואז `npm --prefix server start` — השרת יגיש את ה-build מ-`client/dist`.

## 📊 מקור נתונים

המערכת מנסה למשוך תוצאות אמיתיות ממפעל הפיס (`pais.co.il`). אם המקור אינו זמin בעת ההרצה,
נשמרים נתוני ה-seed הקיימים והמערכת ממשיכה לתפקד. עדכון אוטומטי מתבצע יומית בשעה 23:30 (שעון ישראל)
דרך `node-cron`: משיכת הגרלות חדשות → שמירה → בדיקת התחזית האחרונה מול התוצאה → עדכון ביצועי מודל → תחזית חדשה.

## 🗄️ מבנה הנתונים

טבלאות: `draws`, `predictions`, `prediction_results`, `model_performance`.
קובץ ה-DB נשמר ב-`server/data/lotto.db`.
